# Data Format Samples

## Alert CSV Format (`data/alert_logs/correct_data/`)

These are Snort IDS alert logs with ground-truth labels. Each row represents a single alert.

Columns and example values:

```
id,real_time,sip_str,dip_str,sport,dport,protocol,category_standard,category_standard_id,signature_priority,Label
1,2017-07-03 08:00:05,172.16.0.1,192.168.10.50,52067,80,TCP,Unknown Traffic,3,3,BENIGN
```

| Column | Type | Description |
|--------|------|-------------|
| id | int | Alert ID |
| real_time | datetime | Alert timestamp (YYYY-MM-DD HH:MM:SS) |
| sip_str | string | Source IP address |
| dip_str | string | Destination IP address |
| sport | int | Source port |
| dport | int | Destination port |
| protocol | string | Protocol (TCP/UDP/ICMP) |
| category_standard | string | Snort alert category |
| category_standard_id | int | Category ID (Snort GID:SID format) |
| signature_priority | int | Alert priority/severity |
| Label | string | Ground-truth label (BENIGN or attack type) |

See `samples/alert_sample.csv` for example rows.

## Graph CSV Format (`data/graph_data/`)

Generated by `data_process/log2graph.py`. Each row is an alert subgraph (cluster).

Columns:

| Column | Type | Description |
|--------|------|-------------|
| id | int | Graph ID |
| group_num | int | Louvain community ID |
| edge_index | JSON list | Edge pairs [[src, dst], ...] |
| node_attr | JSON list | Node features [is_center, ip_type_onehot...] |
| edge_attr | JSON list | Edge features [dport_onehot, protocol_onehot, signature_onehot, warn_num] |
| group_attr | JSON list | Graph-level features [node_count, motif_counts...] |
| edge_label | JSON dict | Edge signature counts |
| group_label | JSON set | Set of attack labels in this subgraph |
| nodes | JSON list | IP addresses of nodes |
| start_time | datetime | Time window start |
| end_time | datetime | Time window end |

## Pre-train Triple Format (`data/graph_data/pre-train/`)

Generated by `data_process/generate_pre_train.py`.

Columns: `base_id, match_id, no_match_id`
- `base_id`: anchor graph ID
- `match_id`: positive (same pattern) graph ID
- `no_match_id`: negative (different pattern) graph ID
